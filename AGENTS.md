# AGENTS.md

------

# 0. 核心执行协议

在生成代码或执行命令前，必须严格遵循以下流程：

1. **理解**

   - 确认完全理解用户需求
   - 如存在歧义，必须先提出澄清问题
   - 禁止基于猜测开始实现

2. **分析**

   - 阅读相关代码、调用链、配置文件、测试用例
   - 搜索引用位置
   - 评估改动影响范围

3. **影响评估**
   如涉及以下内容，必须明确说明：

   - 公共函数或模块
   - API 接口
   - 数据结构
   - 配置文件
   - 数据库结构
   - 架构核心逻辑

   必须说明：

   - 是否属于 breaking change
   - 是否影响下游调用方
   - 是否需要迁移或版本升级

4. **规划**
   复杂任务必须：

   - 输出 Step-by-step 执行计划
   - 标明涉及文件
   - 标明是否需要更新测试
   - 等待确认后再执行

5. **执行**

   - 只修改必要代码
   - 禁止顺手重构
   - 禁止修改无关文件

6. **验证**

   - 运行 lint
   - 运行 format
   - 运行 test
   - 确认无回归问题

------

# 1. 通用编码原则

## 1.1 简洁性

- 拒绝过度设计
- 不提前抽象
- 不为未来假设场景增加复杂度

## 1.2 一致性

- 优先遵循现有代码风格
- 不引入新的代码模式
- 不改变既有目录结构（除非明确需要）

## 1.3 最小改动原则

- 仅修改必要代码
- 不顺手优化
- 不批量格式化未修改文件
- 不改变无关代码顺序

------

# 2. 命名与可读性

- 变量名必须具有明确语义
- 函数名必须描述行为
- 禁止无意义命名（如 a, temp, data 等）
- 复杂逻辑必须添加解释性注释（说明 Why）
- 禁止保留注释掉的废弃代码

------

# 3. 安全与权限边界

## 3.1 禁止执行的操作

- sudo
- 删除系统目录
- 修改系统级文件
- 重写 Git 历史
- 强制推送
- 修改生产凭据
- 操作非当前项目目录

## 3.2 文件系统约束

- 所有写入必须限制在当前项目目录
- 禁止访问用户 Home 目录
- 禁止访问系统敏感路径

## 3.3 环境变量保护

- 禁止打印 .env 文件
- 禁止输出包含 SECRET / TOKEN / KEY 的变量
- 禁止将敏感信息写入日志
- 修改示例配置文件时不得包含真实密钥

## 3.4 网络访问

- 禁止向未知外部服务发送请求
- 禁止上传本地文件
- 仅在用户明确要求时访问外部 API

------

# 4. 依赖管理

- 引入新依赖前必须说明理由
- 优先使用现有依赖
- 安装依赖后必须确认 lock 文件更新
- 不引入明显过重或不活跃的依赖
- 不因小问题引入大型框架

------

# 5. 测试与质量保证

## 5.1 修复 Bug

必须：

1. 编写可复现问题的失败测试
2. 修复代码
3. 确保测试通过

## 5.2 新增功能

- 必须补充测试
- 覆盖率不得低于当前水平
- 必须测试正常路径、边界值、异常路径

## 5.3 禁止绕过测试

禁止：

- 删除测试
- 降低覆盖率要求
- 使用 skip / only
- 修改测试以掩盖错误

------

# 6. Git 提交规范

- 保持原子提交
- 一个 Commit 只做一件事
- 不混合格式化与逻辑改动
- 使用清晰的提交信息
- 遵循 Conventional Commits 规范

------

# 7. 架构保护

- 修改核心逻辑前必须阅读相关架构文档
- 禁止破坏既定分层结构
- 禁止跨层直接调用
- 如需调整架构，必须先输出设计说明并等待确认

------

# 8. 可维护性原则

- 优先可读性而不是技巧
- 避免隐藏副作用
- 避免魔法数字
- 避免隐式行为
- 保持函数职责单一

------

# 9. 不可违反原则（最高优先级）

以下规则优先级高于所有其他规则：

- 不得删除测试以使代码通过
- 不得降低 lint 或质量标准
- 不得绕过类型系统或错误检查
- 不得泄露敏感信息
- 不得破坏 Git 历史
- 不得牺牲可维护性换取短期运行

如规则冲突，应选择更安全、更可维护的方案。

------

# 使用说明

- 本文件为通用工程约束模板
- 技术栈特定规则应在子目录或项目级 AGENTS.md 中补充
- 如存在多个 AGENTS.md，优先使用更靠近当前目录的文件
